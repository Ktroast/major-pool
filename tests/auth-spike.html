Claim this account — add email to anonymous user (scenario 3)

Calls supabase.auth.updateUser({ email }). Supabase sends a confirmation email; clicking that link upgrades the anonymous user in place (same user.id, is_anonymous → false).

⚠ Note for PLAN_AUTH.md: linkIdentity({ provider: 'email' }) is OAuth-only in v2. The correct API for adding email to an anonymous user is updateUser({ email }).

Sign in with magic link — cross-device scenario (scenario 4)

Calls supabase.auth.signInWithOtp({ email }). Use this on a second device (or after clearing storage) with the same email used to claim above. Watch the log to see whether Supabase merges identities or produces a new user.
